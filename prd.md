# 网罗众生--人物关系图谱编辑器 - 产品需求文档

---

## 1. 产品概述

**产品名称：** 网罗众生--人物关系图谱编辑器

**核心功能：** 可视化编辑人物关系图谱，支持添加人物/分组/关系、D3.js力导向图实时预览、数据导入导出、云端存储

**技术栈：**
- 前端：纯前端单页应用（HTML + CSS + JavaScript），依赖 D3.js v7
- 后端：Flask（Python）+ SQLite 数据库

**文件结构：**
```
projects/character-network/
├── index.html          # 只读预览页面（可独立部署）
├── editor.html         # 可视化编辑器（主入口）
├── data.json           # 本地导出的数据文件
├── package.json        # 项目配置
├── server.py           # Flask 后端服务
└── schema.sql          # 数据库表结构
```

### 3.13 云端存储（新增）

**功能：** 图谱数据自动同步到服务器数据库，不再仅依赖浏览器 localStorage

**核心价值：**
1. **防丢失** — 浏览器清缓存/重置，数据不丢失
2. **多设备同步** — 同一WiFi下电脑手机都能访问
3. **多图谱管理** — 支持同时管理多个图谱项目

**交互流程：**
- 无需登录，打开即用
- 每次增删改操作后，自动同步到云端（防抖：操作结束后500ms自动保存）
- 页面加载时，优先从云端拉取最新数据，localStorage 作为离线缓存

**底部状态栏（新增）：**
- 显示同步状态图标：🟢已同步 / 🟡同步中 / 🔴离线 / ❌同步失败
- 点击状态图标可手动触发一次同步
- 显示"最后保存: HH:mm"

**模板存储：** 预置模板（红楼、水浒、西游、三国）保存在前端代码中，不上传云端。用户加载模板后生成自己的数据副本再保存到云端。

**API 接口设计：**

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/graphs` | 获取所有图谱列表 |
| GET | `/api/graphs/<graph_id>` | 获取指定图谱的完整数据 |
| POST | `/api/graphs` | 创建新图谱 |
| PUT | `/api/graphs/<graph_id>` | 保存/更新图谱数据 |
| DELETE | `/api/graphs/<graph_id>` | 删除图谱 |

**API 响应格式：**
```json
// GET /api/graphs
{
  "graphs": [
    { "id": "uuid-xxx", "name": "红楼梦", "updated_at": "2026-04-13T10:00:00Z", "node_count": 6, "link_count": 5 }
  ]
}

// GET /api/graphs/<id>
{
  "id": "uuid-xxx",
  "name": "红楼梦",
  "updated_at": "2026-04-13T10:00:00Z",
  "data": { "graphName": "...", "nodes": [], "links": [], "groups": {} }
}
```

### 3.14 我的图谱列表（新增）

**功能：** 支持管理多个图谱，在多个图谱之间切换

**入口：** 编辑面板顶部"📚 我的图谱"按钮

**弹窗内容：**
- 图谱列表：显示所有云端图谱（名称、更新时间、人物数量）
- 每个图谱支持：切换（加载）、重命名、删除
- 支持创建新图谱
- 支持导入本地 JSON 文件到云端

**重命名校验：**
- 名称非空，去除首尾空格
- 最大长度50字符
- 空名称或纯空格 → 提示"图谱名称不能为空"

**删除确认：**
- 删除前弹出确认弹窗："确定删除【图谱名】吗？此操作不可恢复。"
- 确认后同时删除云端和本地缓存

**空状态：** 无图谱时显示引导"还没有图谱，试试加载一个模板吧"

### 3.15 本地存储

**存储 Key：** `characterNetworkData`（降级为离线缓存）

**存储数据结构：**
```json
{
  "graphId": "uuid-xxx",     // 当前图谱ID
  "graphName": "红楼梦",
  "data": { ... },            // 图谱完整数据
  "updatedAt": "2026-04-13T10:00:00Z"  // 本地更新时间戳
}
```

**自动保存时机：**
- 创建新图谱后
- 加载模板后
- 添加/删除人物后
- 添加/删除关系后
- 添加分组后
- 修改图谱名称后
- 编辑人物或关系后

**页面加载逻辑：**
1. 检查 localStorage 是否有数据
2. 有 → 比较 `localStorage.updatedAt` vs `云端.updatedAt`
   - localStorage 更新 → 加载本地，显示"本地版本"
   - 云端更新 → 拉取云端，加载云端版本
   - 两者都有更新（时间相近）→ 以云端为准
3. 无 → 从云端拉取
4. 云端无数据 → 创建空白图谱（默认分组），引导加载模板

**删除图谱时：** 同时清除 localStorage 中该图谱的缓存

### 3.16 同步失败处理（新增）

**自动重试：**
- 同步失败后自动重试，最多3次
- 每次重试间隔2秒
- 重试期间状态显示 🟡同步中
- 3次都失败 → 显示❌同步失败，不再自动重试

**手动恢复：**
- 用户可点击❌图标手动重试同步
- 网络恢复后自动检测并同步

## 6. 数据库设计（Flask + SQLite）

### 6.1 表结构

```sql
-- 图谱主表
CREATE TABLE graphs (
    id TEXT PRIMARY KEY,          -- UUID
    name TEXT NOT NULL,           -- 图谱名称
    data TEXT NOT NULL,           -- JSON字符串（完整的图谱数据）
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### 6.2 Flask 服务配置

- 数据库文件：`data.db`（SQLite）
- 服务端口：默认 `5000`
- CORS：允许前端 `editor.html` 直连

## 8. 已知限制

1. **不支持撤销/重做**：删除操作不可恢复
2. **localStorage 容量限制**：约5MB，离线缓存大型图谱可能溢出
3. **无用户权限管理**：单用户本地使用，无需登录
